# Analysis of the 2017 Equifax Data Breach

## Overview

This project presents a comprehensive analysis of the **2017 Equifax Data Breach**, one of the most significant cybersecurity incidents in history. The breach exposed the personal information of approximately **147 million individuals** after attackers exploited a known vulnerability in the **Apache Struts web application framework (CVE-2017-5638)**.

The project examines the technical causes of the breach, its impact on individuals and organizations, the failures in security management, and the cybersecurity lessons that can help prevent similar incidents in the future.

---

## Objectives

- Understand the root cause of the Equifax data breach.
- Analyze the exploited vulnerability (CVE-2017-5638).
- Examine the security failures that contributed to the attack.
- Assess the financial, legal, and reputational impact.
- Identify cybersecurity best practices for preventing similar incidents.

---

## Background

Equifax is one of the largest consumer credit reporting agencies in the world. In 2017, attackers gained unauthorized access to Equifax systems by exploiting an **unpatched Apache Struts vulnerability**.

The attackers remained undetected for several weeks, allowing them to access and steal highly sensitive consumer information, including:

- Full Names
- Social Security Numbers
- Dates of Birth
- Addresses
- Driver's License Numbers
- Credit Card Information
- Personal Identification Data

The incident highlighted serious weaknesses in vulnerability management, security monitoring, and incident response.

---

## Root Cause Analysis

The primary causes of the breach included:

- Failure to patch a known critical vulnerability (CVE-2017-5638)
- Weak vulnerability management practices
- Delayed detection of malicious activity
- Inadequate network monitoring
- Poor security governance
- Insufficient incident response procedures

---

## Security Failures Identified

This analysis identified several cybersecurity failures, including:

- Unpatched Apache Struts server
- Poor asset inventory management
- Weak network segmentation
- Inadequate encryption practices
- Lack of continuous vulnerability scanning
- Ineffective Security Information and Event Management (SIEM)
- Delayed incident detection and response

## Impact of the Breach

### Technical Impact

- Compromise of sensitive consumer records
- Unauthorized access to internal systems
- Large-scale data exfiltration

### Business Impact

- Significant financial losses
- Regulatory penalties
- Multiple lawsuits
- Loss of customer trust
- Reputational damage

### Security Impact

- Increased identity theft risks
- Credit fraud
- Long-term privacy concerns

---

## Cybersecurity Lessons Learned

The Equifax breach demonstrates the importance of:

- Timely patch management
- Continuous vulnerability assessments
- Regular penetration testing
- Network segmentation
- Security monitoring using SIEM solutions
- Multi-factor authentication (MFA)
- Data encryption
- Security awareness training
- Effective incident response planning
- Strong cybersecurity governance

---

## Recommended Security Controls

To reduce the likelihood of similar incidents, organizations should implement:

- Patch Management Program
- Vulnerability Scanning
- Penetration Testing
- Network Segmentation
- Continuous Security Monitoring
- Endpoint Detection and Response (EDR)
- Security Information and Event Management (SIEM)
- Multi-Factor Authentication (MFA)
- Encryption of Sensitive Data
- Incident Response Plan
- Disaster Recovery Plan
- Employee Security Awareness Training
